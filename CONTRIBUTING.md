# Contributing

Thanks for your interest in PortHound.

## Ground rules
- Use this tool only for authorized security work.
- Keep changes focused and small when possible.
- Avoid adding heavy dependencies without discussion.

## How to contribute
1. Fork the repo and create a branch: `feat/your-change` or `fix/your-change`.
2. Make your changes with clear, descriptive commits.
3. Update docs if behavior changes.
4. Open a pull request with a short summary and test notes.

## Development
- Backend: `python manage.py`
- Frontend: `cd frontend && npm install && npm run serve`

## Reporting issues
Use the issue templates and include:
- OS / Python / Node versions
- Steps to reproduce
- Expected vs actual behavior

## Security
Please do not open public issues for vulnerabilities.
See `SECURITY.md`.
